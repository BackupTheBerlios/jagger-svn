Jagger Homepage

Overview

Jagger is a Java application monitoring tool using JMX technology to aggregate, archive and visualize monitoring data for larger computer clusters, giving developers and administrators both a succinct and comprehensive view into their systems, which normal JMX consoles cannot do due to information overflow.

Major features

  • implemented in Groovy — flexible, yet fully integrated into J2EE environents.
  • console mode with tailored information panels about the state of your JVM, container and applications.
  • description of monitoring data model (PLANNED)
  • aggregation of distributed mbean data in clustered host setups, exporting the result as mbeans again (PLANNED)
  • archival of monitoring data (PLANNED)
  • SLA configuration, monitoring and enforcement (PLANNED)

Usage szenarios and system environment of Jagger.

Current limitations

  • only tested for JVMs that run Tomcat monitored by the JavaServiceWrapper (need feedback from other environments).
  • no support for other application servers than Tomcat 5.5 yet (again, need help or feedback).
  • since the start script handles the activation of the console RAW mode, Jagger is slightly UNIX-dependant (How do I handle raw keyboard I/O in Java under Windows?)
  • version reporting uses WEB.DE standards (see applicationContext.xml for an example, you might find that useful for your projects).

How to get started

First, get a source or binary (pre-built) release from the download pages. After installing (see also INSTALL.txt), you can directly start by calling the "jagger" command like this: 

jagger -n lxjavatest9101:28113 -u monitor -w monitor

Check out the command line page to learn about available command line options. For more information on how to better adapt Jagger to your environment, see the configuration page.
